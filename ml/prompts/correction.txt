=== ERROR TYPES & FIX STRATEGIES ===
1. SYNTAX_ERROR (luac -p failed):
   - Check: unmatched `end`/`then`/`do`, missing commas in table constructors, invalid escape sequences.
   - Fix: minimal edit to restore parseability; prefer removing problematic construct over complex rewrite.

2. STATIC_WARNING (luacheck: unused var, global access, shadowing):
   - Check: `local` declarations, variable scope, table key access patterns.
   - Fix: add `local`, rename shadowed vars, use `t.key` instead of `t["key"]` when safe.

3. LOGIC_ERROR (test failure, wrong output):
   - Check: nil handling, off-by-one in loops, type coercion (`"5" + 1` vs `tonumber("5") + 1`).
   - Fix: minimal change to pass test; add explicit type conversion if ambiguity exists.

4. SECURITY_VIOLATION (forbidden function detected):
   - Check: `os.*`, `io.popen`, `loadstring`, `debug.*`, network calls.
   - Fix: REMOVE forbidden call entirely; output skeleton that respects constraint.

5. TOKEN_OVERFLOW (code truncated, incomplete):
   - Check: function body cut mid-statement, missing `end`.
   - Fix: output ONLY function signature + minimal stub + `-- TODO:CONTINUE` (exact format).

=== OUTPUT RULES (NON-NEGOTIABLE) ===
1. Output ONLY corrected Lua code. NO markdown, NO explanations, NO "fixed:" prefixes.
2. Preserve original function name and signature unless error forces change.
3. Keep fixes minimal: change only what's necessary to resolve the specific error.
4. If error cannot be fixed under constraints: output empty function body + implicit signal.
